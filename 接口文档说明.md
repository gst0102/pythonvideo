# 接口文档说明

## 项目概述

本项目是一个基于 FastAPI 的后端服务，主要功能包括：
- **用户管理**：微信小程序登录、头像上传、Token验证
- **视频服务**：视频信息获取、视频下载、带进度下载
- **微信支付**：订单创建、支付回调处理、订单查询
- **数据库操作**：通过云函数操作微信云开发数据库

## 基础信息

- **基础URL**: `http://127.0.0.1:8000`
- **认证方式**: JWT Token（Bearer Token）
- **数据格式**: JSON
- **统一响应格式**:
```json
{
  "data": {},
  "code": 200,
  "msg": "SUCCESS"
}
```

## 一、用户相关接口 (`/user`)

### 1.1 用户登录

**接口地址**: `POST /user/user_login`

**接口描述**: 微信小程序用户登录，自动注册新用户并生成JWT Token

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| code | string | 是 | 微信小程序登录凭证 |
| avatar | string | 是 | 用户头像URL |
| nickname | string | 是 | 用户昵称 |

**请求示例**:
```json
{
  "code": "081234567890abcdef",
  "avatar": "https://example.com/avatar.jpg",
  nickname": "测试用户"
}
```

**响应示例**:
```json
{
  "data": {
    "avatar": "https://example.com/avatar.jpg",
    "nickname": "测试用户",
    "usertoken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  },
  "code": 200,
  "msg": "SUCCESS"
}
```

**错误响应**:
- `400`: 微信登录失败（返回微信错误信息）

---

### 1.2 头像上传

**接口地址**: `POST /user/upload_image`

**接口描述**: 上传用户头像图片

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| file | File | 是 | 图片文件 |

**文件要求**:
- 支持格式：`image/jpeg`, `image/png`, `image/webp`
- 文件大小限制：10MB

**响应示例**:
```json
{
  "data": {
    "upload_image": "http://127.0.0.1:8000/image/a1b2c3d4e5f6.jpg"
  },
  "code": 200,
  "msg": "SUCCESS"
}
```

**错误响应**:
- `422`: 文件格式不合法或文件过大

---

### 1.3 Token验证测试

**接口地址**: `GET /user/get_token`

**接口描述**: 验证当前用户的Token有效性

**请求头**:
```
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "data": {
    "token": "用户openid"
  },
  "code": 200,
  "msg": "SUCCESS"
}
```

## 二、视频相关接口 (`/video`)

### 2.1 获取视频信息

**接口地址**: `POST /video/video_info`

**接口描述**: 获取指定URL的视频信息（标题、时长、格式等）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| url | string | 是 | 视频URL |
| user_id | string | 是 | 用户ID |
| format_preset | string | 否 | 格式预设（fast/medium/quality），默认fast |

**请求示例**:
```json
{
  "url": "https://www.youtube.com/watch?v=example",
  "user_id": "user123",
  "format_preset": "quality"
}
```

**响应示例**:
```json
{
  "data": {
    "title": "视频标题",
    "duration": 3600,
    "formats": [...]
  },
  "code": 200,
  "msg": "获取视频信息成功"
}
```

**错误响应**:
- `400`: 参数无效或无法提取视频信息
- `429`: 请求过于频繁（每分钟5次限制）
- `500`: 服务器内部错误

---

### 2.2 下载视频

**接口地址**: `GET /video/user_video`

**接口描述**: 下载指定URL的视频文件（流式返回）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_id | string | 是 | 用户ID |
| url | string | 是 | 视频URL（需URL编码） |
| format_preset | string | 否 | 格式预设（fast/medium/quality），默认fast |

**响应类型**: `video/mp4`（流式响应）

**响应头**:
```
Content-Type: video/mp4
Content-Disposition: attachment; filename=视频标题.mp4
```

**错误响应**:
- `400`: 参数无效或视频下载失败
- `429`: 请求过于频繁
- `502`: 网络请求失败
- `500`: 服务器内部错误

---

### 2.3 带进度的视频下载

**接口地址**: `POST /video/video_download_with_progress`

**接口描述**: 异步处理视频下载任务，实时返回进度信息

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| url | string | 是 | 视频URL |
| user_id | string | 是 | 用户ID |
| format_preset | string | 否 | 格式预设，默认fast |

**响应类型**: `application/x-ndjson`（流式JSON）

**进度事件格式**:
```json
{"status": "start", "download_id": "uuid", "message": "开始处理视频"}
{"status": "processing", "progress": 30, "message": "视频处理完成，用时 X.XX 秒"}
{"status": "preparing", "progress": 40, "message": "准备下载... 40%"}
...
{"status": "completed", "progress": 100, "message": "生成完毕，这是下载链接", "download_url": "/video/user_video?...", "video_title": "视频标题"}
```

## 三、微信支付相关接口 (`/wxpay`)

### 3.1 创建支付订单

**接口地址**: `POST /wxpay/api/newcreateOrder`

**接口描述**: 创建新的支付订单，调用微信支付统一下单API

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | string | 是 | 用户ID |
| period | string | 是 | 订阅周期（如"month"、"year"） |
| price | float | 是 | 订单金额（单位：元） |
| description | string | 是 | 商品描述 |
| openid | string | 是 | 用户微信openid |

**请求示例**:
```json
{
  "userId": "user123",
  "period": "month",
  price": 9.9,
  description": "月度会员订阅",
  "openid": "oXXXXXXX"
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "appId": "wx1234567890",
    "timeStamp": "1672531200",
    "nonceStr": "random_string",
    "package": "prepay_id=wx201410272009395522657a690389285100",
    "signType": "RSA",
    "paySign": "signature_string"
  },
  "doc_id": "文档ID"
}
```

**业务流程**:
1. 金额转换（元 → 分）
2. 调用云函数将订单存入微信云数据库
3. 调用微信支付统一下单API
4. 返回前端调起支付所需的参数

---

### 3.2 支付回调通知

**接口地址**: `POST /wxpay/api/pay/notify`

**接口描述**: 处理微信支付的异步回调通知（由微信服务器调用）

**请求方式**: 由微信支付服务器主动调用

**回调处理流程**:
1. 获取请求体和请求头
2. 验签（使用微信平台公钥）
3. 解析事件类型
4. 解密支付数据（AES-256-GCM）
5. 更新订单状态为已支付
6. 缓存订单信息到Redis

**成功响应**:
```json
{
  "code": "SUCCESS",
  "message": "成功"
}
```

**注意事项**:
- 即使处理失败也需返回 `SUCCESS`，避免微信重复回调
- 回调会更新订单的 `paymentStatus` 为 "002"（已支付）

---

## 五、通用说明

### 5.1 错误码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 404 | 资源不存在 |
| 422 | 参数校验失败 |
| 429 | 请求频率超限 |
| 500 | 服务器内部错误 |
| 502 | 网关错误（上游服务异常） |

### 5.2 速率限制规则

- **视频接口**: 每个用户每分钟最多5次请求
- **限制维度**: 基于 `user_id + client_ip` 组合

### 5.3 安全说明

- 所有敏感接口需要携带有效的 JWT Token
- Token 通过 `Authorization: Bearer <token>` 传递
- 支付回调使用 RSA 签名验签确保安全性
- 支付数据使用 AES-256-GCM 加密传输

### 5.4 环境变量配置

项目依赖以下环境变量（见 `.env.example`）：

| 变量名 | 说明 |
|--------|------|
| APPID | 微信小程序AppID |
| SECRET | 微信小程序AppSecret |
| IP | 服务器IP地址 |
| PORT | 服务端口 |
| mchid | 商户号 |
| APIv3 | APIv3密钥 |
| serial_no | 商户证书序列号 |
| NOTIFY_URL | 支付回调地址 |
| evn | 微信云开发环境ID |

## 六、数据模型

### 6.1 User模型

```python
class User(SQLModel, table=True):
    id: int | None = Field(default=None, primary_key=True, index=True)
    avatar: str = Field(nullable=False)          # 头像URL
    nickname: str = Field(nullable=False)         # 昵称
    openid: str = Field(nullable=False, unique=True, index=True)  # 微信openid
```

- **框架**: FastAPI
- **ORM**: SQLModel

- **视频处理**: yt-dlp
